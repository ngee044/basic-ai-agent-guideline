# CLAUDE.md

이 저장소는 언어별 개발 가이드라인 "템플릿 모음"입니다. 특정 프로젝트에 종속되지 않는 범용 가이드라인/코드 규약/ISO 문서 규약을 담고 있으며, 실제 프로젝트에 복사·수정해 사용합니다.

## 이 저장소를 다룰 때의 지침

- 사용자가 새 프로젝트 가이드라인을 요청하면, 대상 언어 디렉터리(`golang_guideline/`, `python_guideline/`, `cpp_guideline/`)를 기준으로 복사·수정해 제공합니다.
- 각 언어 디렉터리는 `CLAUDE.md`, `CODING_CONVENTION.md`, `ISO_DOCUMENTS_CONVENTION.md` 세 파일로 구성됩니다. 새 언어 가이드라인을 추가할 때도 이 레이아웃을 그대로 따릅니다.
- 문서 본문은 한국어로 작성하되, 기술 용어·라이브러리명·코드 식별자·CLI 명령어는 원문(영어) 그대로 유지합니다.
- 프로젝트별로 채워야 하는 값은 `{{PLACEHOLDER}}` 형식으로 표기합니다(예: `{{PROJECT_NAME}}`, `{{MODULE_PATH}}`, `{{DB}}`).
- 코드 예시는 해당 언어로 실제 동작하고 관용적(idiomatic)이어야 하며, 확실하지 않은 API/문법은 넣지 않습니다.

## 전역 원칙(global_guideline/CLAUDE.md 준수)

`global_guideline/CLAUDE.md`의 전역 원칙을 따릅니다.

- 한국어·존댓말로 답변/작성하고, 불필요한 서론/요약은 생략합니다.
- 임의로 커밋/PR을 생성하지 않습니다(사용자가 직접 수행). 단, 사용자 요청에 따른 행동은 유효합니다.
- 모든 프로젝트 분석은 ISO 기반 문서로 작성하고, 적합성·추적성을 확보합니다. 문서는 프로젝트 하위 `docs/` 폴더에 저장합니다.
- 주석은 WHY가 비자명할 때만 작성합니다.
- 추측하지 않습니다. 모르면 검색·확인 후 답변하며, 지시가 불명확하면 되물어봅니다.
- 컴파일/타입체크가 가능한 언어는 반드시 빌드·검사하여 에러 0을 확인합니다.
